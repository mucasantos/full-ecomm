Implementando o UsersRepository
O Repository Pattern é baseado na ideia de que o código de acesso a dados deve ser separado 
do código de lógica de negócios. O Repository Pattern fornece uma maneira de gerenciar 
o código de acesso a dados em um local centralizado, reduzindo assim a duplicação de código 
e melhorando a capacidade de manutenção do código.
